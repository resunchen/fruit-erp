### 开始开发工作（每周重复）
```
1. 打开当周的 TODO 文档
   └─ TODO/TODO_P1_Week[N]_[模块名]_[N].md

2. 理解本周的功能清单

   └─ 后端 API → 前端界面 → 数据库 → 测试

4. 每完成功能点就勾选
   └─ - [x] 已完成的功能

5. 定期 Git 提交
   └─ git commit -m "feat: 完成的功能点"

6. 周末完成总结
   └─ 更新 TODO 的"完成情况"部分
```

## 🚨 重要提示

### 开发前必读
1. **阅读规范** - 在写代码前必须读 DEVELOPMENT_STANDARDS.md
2. **理解 TODO** - 每周开始前完整阅读当周 TODO 文档
3. **遵循结构** - 按照项目文件夹结构组织代码
4. **类型安全** - 使用 TypeScript，不允许 `any` 类型

### 常见误区
❌ **不要**：直接开始写代码而不了解设计
❌ **不要**：跳过规范文档自我发挥
❌ **不要**：混乱地提交 commit，不遵循规范
❌ **不要**：一周内一次性完成所有功能再提交

✅ **应该**：先读文档，理解设计，再按规范开发
✅ **应该**：每完成 2-3 个功能点就提交一次
✅ **应该**：遇到问题时参考 QUICK_REFERENCE.md

## 📞 常见问题秒答

**Q: 我是开发人员，现在应该做什么？**
A:
1. 读 [README_DEV_GUIDE.md](./README_DEV_GUIDE.md)
2. 读 [DEVELOPMENT_STANDARDS.md](./DEVELOPMENT_STANDARDS.md)
3. 打开 `TODO_P1_Week1_项目初始化_01.md`
4. 开始按功能清单逐项完成

**Q: 项目太大了，我不知道从哪开始？**
A: 别急！按以下步骤：
- 第 1 周：完成项目初始化和基础框架
- 第 2 周：实现采购管理模块
- 第 3 周：实现仓储和发货模块
- 第 4 周：实现成本核算和集成测试
每周的 TODO 都很清楚，逐项完成就行。

**Q: 代码规范那么多，记不住怎么办？**
A: 不需要全部记住！
- 开发时参考 [QUICK_REFERENCE.md](./QUICK_REFERENCE.md)（快速查询表）
- 每个功能点实现时参考对应的代码模板
- 代码审查时 reviewer 会指正，逐次改进

**Q: 如果某个功能点花的时间比预期久？**
A: 很正常！这时候：
1. 在 TODO 中更新实际耗时
2. 记录遇到的问题和解决方案
3. 与 lead 沟通是否需要调整计划
4. 继续完成其他功能点

**Q: 完成的 TODO 应该保留还是删除？**
A: 保留并归档！
1. 完成后移到 `TODO/COMPLETED/` 文件夹
2. 保留完整记录便于查询和学习
3. 形成项目知识库
