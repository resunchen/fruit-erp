## 🎯 关键信息速查

### 项目规模
- **总周期**: 12 周
- **前端框架**: React 18 + Tailwind CSS + shadcn/ui
- **后端框架**: Node.js + Express
- **数据库**: Supabase (PostgreSQL)
- **核心模块**: 13 个（采购、仓储、发货、成本、追溯等）

### 优先级分布
| 优先级 | 周次 | 模块数 | 重要性 |
|-------|------|--------|--------|
| P1 | 1-4 周 | 5 个 | ⭐⭐⭐ 必须完成 |
| P2 | 5-8 周 | 5 个 | ⭐⭐ 重要功能 |
| P3 | 9-10 周 | 4 个 | ⭐ 补充功能 |
| P4 | 11-12 周 | 4 个 | ⭐ 高级功能 |

### 关键里程碑
- **第 4 周末**: 核心业务基础版本（采购→仓储→发货）
- **第 8 周末**: 管控功能完整版本（追溯、物料、通知）
- **第 10 周末**: 全功能初步版本（客户、人员、运输）
- **第 12 周末**: 正式上线版本（权限、素材、分析）

### 非功能性要求
- 页面加载时间 ≤ 2 秒
- 并发用户 ≥ 50
- 系统可用性 ≥ 99.5%
- 代码测试覆盖率 ≥ 70%

## 📞 常见问题

**Q: 应该按什么顺序阅读文档？**
A: 见上面"推荐阅读顺序"部分。初次接触项目建议按 README_DEV_GUIDE → DEVELOPMENT_STANDARDS → DEVELOPMENT_PLAN 的顺序。

**Q: 如果某周有太多功能，怎么办？**
A: 这是正常的。按 TODO 文档中的顺序逐项完成，遇到困难可以与 lead 沟通是否需要拆分。

**Q: 代码规范太复杂，应该怎么学？**
A: 先读代码规范文档的"代码模板"部分，然后参照模板编写。代码审查时 reviewer 会指正。

**Q: 是否必须全部阅读所有文档？**
A: 不需要。根据你的角色选择相关部分：
- 开发人员：必读 DEVELOPMENT_STANDARDS 和当周 TODO
- 项目经理：重点读 DEVELOPMENT_PLAN 和每周 TODO
- Architect：完整阅读所有文档
