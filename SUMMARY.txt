✅ 开发规范要点（一句话总结）：

- 每次开发前：创建 TODO 文档，列出 20-40 个具体功能点
- 开发过程中：每完成功能点就勾选复选框和提交 Git commit
- 开发完成后：更新 TODO 的完成情况，归档到 COMPLETED 文件夹
- 代码标准：遵循 TypeScript、ESLint、命名规范、项目结构
- 提交规范：使用规定的 commit message 格式
- 测试要求：覆盖率 ≥ 70%，所有测试通过
- 文档要求：API 文档、代码注释、功能说明完整
