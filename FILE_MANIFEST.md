**Q: 应该按什么顺序阅读文档？**
A: 见上面"使用指南"部分。根据你的角色选择。

**Q: 所有文档都是必读吗？**
A: 不是。开发人员必读 STANDARDS 和当周 TODO。
   其他文档根据需要查询即可。

**Q: 怎样追踪开发进度？**
A: 通过 TODO 文档中的复选框。每完成一项就勾选。
   周末汇总统计，更新"完成情况"部分。

**Q: 如果文档内容有错误或不清楚？**
A: 及时反馈并更新。这是活的文档，会不断改进。

**Q: 新的迭代（Week5+）如何处理？**
A: 参考 Week1-4 的 TODO 文档格式，创建新的 TODO。
   或者在完成 Week4 后，我会帮你创建后续 TODO。
